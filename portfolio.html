Reflection

When I was first introduced to this project I was given the option to create story or a game and I decided to create a story of my life and allowing the users to control the decisions of my life. I chose to do a story because since I was a young kid I have had goals set for myself so I tried to incorporate the goals into this story. I chose to put Mr. Engstrom in my story since he has helped me choose the decisions I have made throughout my highschool career and he introduced me to engineering and actually made me believe I can pursue engineering in the future.

How this story works is you will startat a title page with the title of the story, mine and my parteners name, a picture of my face which is your character, and a button to begin the story. Once you begin you will be told the story line and the instructions on how to move and make decisons. While making the first decision you have the option to get advice, you can then make the first decsion, once that is made the backdrop will change and depending on what decsion you make determines what backdrop you go to next. If you make the correct decisions you will end up at the beach living a good life and can then press the space bar and money will fall from the top of the screen, if you make a wrong decision then you will end up working at mcdonalds for the rest of your life. Once you're done you then finish by clicking the green button in the top corner.



All around this project was a great learning experience and allowed me to practice coding with scratch and also helped me learn about more detailed topics such as algorithms and abstractions. In this case the algorithm and abstraction are the same piece of code. The algorithm and abstraction in my story are the user controls that allows for the user or player to move the directional arrows to guide the character to make the decisions. This is considered the algorithm because it allows the sprite to move up, down, and side-to-side with one loop of code, this code is used throughtout the entire story when the user has to move the sprite and by using this algorithm in a forever loop, it makes the code so much simpler. This piece of code can also be considered the abstraction because I used all of the user controls in one forever loop which allowed the code to be very simple and it is used throughout the entire story when the sprite has to move.




The two major difficulties we had was making it so once the space bar is pressed the money will fall on background four (at the beach), and the other one was making sure the correct backdrop appeared when each specific decision was made. One of the most useful things I learned while creating this story has to be using the "say" text boxes because it was very helpful when including the instructions to the story, and also letting the user know the story line as the story continues. At first I was including the story line just by using the txt on each backdrop which made every one of the backdrops look very cluttered, unorganized, and in some spots, too hard to read.

If I had to do the story over again I do not think I would change very much of the story line, but I would try and make it more appealing to the users, allow for the user to be more involved and making more detailed decisions, maybe even some decisions that affects other decisions, and then including character select. Character select is the main feature I regret not applying to this story. This project was an overall great experience and taught me alot including taking every sprint seriously, managing my time properly, and even communication skills either with my teacher, fellow classmates, and even my partner.

Image Manipulation Project

Original Picture and Manipulated Picture


During this project my partner and I came up with this code using the tutorial provided, other online sources, and our own basic python knowledge to create this image manipulation code. In this case, our code takes a photo that we have updated into our PIL image library and imports it to the program along with set preprogrammed filters, it splits the RGB color coordinants to then only provide the red portion of RGB which allows the image to appear pink/red. The code then rotates the image 180 degrees and gives it a slight preprogrammed blur effect. It will then show you the original image, the manipulated image, and save the new image as a seperate file to your set directory.

During this project my partner and I learned there are endless possibilities you can do to images using python. You can do something as simple as turning the picture to greyscale all the way to changing each pixel in the image to a different color of you wanted too. We also learned that there are preprogrammed features inside python that do many different things for example the filter we used to blur the image.

My partner Jose and I decided to do our malware presentation on the Ping Of Death. The ping of death is simply multiple computers working simultaniously to infect a single computer with a virus/ overload onto the computer kicking the user offline, making it difficult for one to get back online. Ping Of Death was popular while playing videogames, when playing online a hacker could "Ping" you to death kicking you offline although they are not near you. Simulation Modification

This simulation modification was intended to inform us about simulations and how they work to provide for real life situations. Simulations are used to replicate situations that are far too complicated and or expensive. The simulations give real time answers for the imitations created. The simulation we chose to modify as a group was taking the two colors (Blue and Green), which represents atoms, split in the middle, and alter the code to allow for there to be four colors (Green, Blue, Red, and Pink) divided into four quadrants. The simulation was used to find max diffusion distance between atoms. Our modification was intended to simulate adding more atoms to see if it effected the max distance each atom travels. This simulation was created on netlogo which is a simulation program. My partner and I learned what simulations are and that they are difficult to code/create, but they save money, time, and resources. Creating a simulation on the computer or on a smaller scale is much easier and more effective than doing the actual problem multiple times before it is successful.
